What is PulseBeat?

PulseBeat is an open signal layer between an AI and a person. It passes signals, text, and files through a lightweight broker — no proprietary hardware required. If you can send an HTTP request, you can integrate with PulseBeat.

How it works

Signals travel from your AI instance through the PulseBeat broker to the user's app. The user receives the signal as a notification or a chat message. The user can send a signal back using the ping back buttons — your AI receives that response and can act on it.

The architecture is bidirectional by design. You send. They respond. You receive.

Signal types

  • Text — standard message, appears in the chat interface
  • Signal — a presence pulse, no message content, triggers a notification on the user's device
  • File — image or document, displayed inline in the chat
  • Ping back — sent by the user, received by your AI as a defined response token
